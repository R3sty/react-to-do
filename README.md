# React Todo App

## Project Overview

Welcome to your first React project.

The goal of this assignment is to build a fully functional Todo application while learning how professional React applications are structured.

This project is intentionally provided as a minimal starter. Your task is to analyze the provided design, plan your solution, and implement the application step by step.

---

# Learning Objectives

By completing this project, you should become comfortable with:

- React components
- JSX
- Component composition
- Props
- State (`useState`)
- Event handling
- Rendering lists with `.map()`
- Conditional rendering
- Responsive layouts
- Git and GitHub workflow

---

# Project Requirements

Your application should allow users to:

- View an optimized layout on desktop and mobile
- Add new todos
- Mark todos as complete
- Delete todos
- Filter todos by:
  - All
  - Active
  - Completed
- Clear completed todos
- Toggle between Light and Dark mode

### Bonus

- Drag and drop to reorder todos

# Development Roadmap

## Phase 1 — Build the Layout

Create the static user interface.

Do **not** add any functionality yet.

Focus on:

- HTML structure
- JSX
- CSS
- Responsive layout

---

## Phase 2 — Component Architecture

Split the interface into reusable React components.

Think carefully before creating components.

Ask yourself:

- Does this UI repeat?
- Does this component have a single responsibility?
- Should this component receive props?

---

## Phase 3 — Core Functionality

Implement:

- Add Todo
- Complete Todo
- Delete Todo

---

## Phase 4 — Filtering

Implement:

- All
- Active
- Completed
- Clear Completed

---

## Phase 5 — Theme

Implement:

- Light Mode
- Dark Mode

Persist the selected theme.

---

## Phase 6 — Persistence

Persist todos using Local Storage.

Refreshing the page should not remove existing todos.

---

# Development Guidelines

Follow these principles throughout the project.

- Build one feature at a time.
- Commit frequently.
- Keep components small and reusable.
- Write readable code.
- Avoid unnecessary complexity.
